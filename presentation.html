Présentation générale

Application web complète de gestion d'établissement secondaire. Elle centralise la gestion des élèves, classes, notes, absences, et génère automatiquement les documents administratifs (bulletins, convocations, tableaux d'honneur).

🎓

267 élèves

18 classes de la 6e à la Tle, avec séries (ALL, ESP, CHS, A, C, D, TI)

📊

Notes & Bulletins

Saisie individuelle ou groupée, calcul automatique des moyennes et rangs

📄

Documents PDF & Excel

Bulletins trimestriels/annuels, convocations, tableaux d'honneur, exports XLSX

🔔

Notifications SMS

Alertes absences et bulletins via Twilio (ou simulation)

🌙

Mode sombre

Thème clair/sombre avec persistance localStorage et détection système

📈

Dashboard interactif

Graphiques Chart.js, statistiques en temps réel, absentéisme par classe

Architecture MVC

L'application suit strictement le pattern Modèle-Vue-Contrôleur avec Jakarta EE (Servlets + JSP + JSTL).

Couches de l'application

  • Modèle — POJO : Eleve, NoteEleve, Absence, Classe, Utilisateur
  • Vue — JSP dans /WEB-INF/vues/ (17 templates)
  • Contrôleur — 15 Servlets avec @WebServlet
  • DAO — 7 interfaces + implementations (JDBC pur)
  • Service — PdfService, ExcelService, SmsService, NotificationService
  • DTO — EleveSearchCriteria, NoteSearchCriteria

Avantages de l'architecture

  • Séparation stricte des responsabilités
  • Sécurité : JSP non accessibles directement (dans WEB-INF)
  • DAO avec try-with-resources et PreparedStatement
  • Filtre d'authentification centralisé (AuthFilter)
  • Rôles et permissions granulaires (Admin/Censeur/Surveillant)

Requêtes clés

Moyenne pondérée

SELECT SUM(notes_valeur * coefficient) / SUM(coefficient)
FROM note_eleve WHERE eleve_id=? AND trimestre=?

Rang avec window function

SELECT RANK() OVER (ORDER BY SUM(n*coef)/SUM(coef) DESC)
FROM note_eleve WHERE trimestre=? GROUP BY eleve_id

Mesures de sécurité

  • Mots de passe : hashés avec JBCrypt ($2a$)
  • Session : timeout 30 minutes, attributs sécurisés
  • AuthFilter : @WebFilter("/app/*") vérifie session + rôle
  • Injection SQL : PreparedStatement partout
  • JSP protégées : dans /WEB-INF/, non accessibles directement
  • XSS : c:out pour l'échappement HTML
  • Headers HTTP : X-Frame-Options, CSP, X-Content-Type-Options

Déploiement

Prérequis

  • Java 25 (JDK)
  • Apache Maven 3.9+
  • Apache Tomcat 11.0.21
  • MySQL 8.4 + base de données créée

Étapes de déploiement

# 1. Créer la base et importer le schéma
mysql -u root -p < schema.sql

# 2. Configurer db.properties
echo "db.url=jdbc:mysql://localhost:3306/lycee" > db.properties
echo "db.user=root" >> db.properties
echo "db.password=monpass" >> db.properties

# 3. Builder le WAR
mvn clean package -DskipTests

# 4. Déployer sur Tomcat
cp target/lycee.war /opt/tomcat/webapps/

# 5. Démarrer Tomcat
/opt/tomcat/bin/startup.sh

Configuration Tomcat

# setenv.sh (dans CATALINA_BASE/bin)
export CATALINA_OPTS="$CATALINA_OPTS \
  -DTWILIO_ACCOUNT_SID=... \
  -DTWILIO_AUTH_TOKEN=... \
  -DTWILIO_PHONE_NUMBER=+1234567890"
